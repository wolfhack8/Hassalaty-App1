# Landing-page mockups

Drop your screenshots here with these **exact** filenames. Until you add them,
the landing shows a clean branded placeholder in each frame (no broken images).

| File | What to capture | Frame | Best size |
| --- | --- | --- | --- |
| `dashboard.png` | **Dashboard** — English, light mode | Browser (16:10) | ~1440×900 |
| `dashboard-ar.png` | **Dashboard** — Arabic (RTL), light mode | Browser (16:10) | ~1440×900 |
| `markets.png` | **Markets** screen — English | Browser (16:10) | ~1440×900 |
| `assistant.png` | **AI Assistant** screen — English | Browser (16:10) | ~1440×900 |
| `card.png` | **Virtual Card** screen — portrait / mobile | Phone (9:19) | ~390×844 |

Tips:
- Browser-frame images are cropped `object-cover` from the top — capture the
  full app viewport (sidebar + content) and let the bottom crop if needed.
- For `card.png`, take a narrow/mobile-width screenshot so the wallet stack fits
  the phone frame nicely.
- PNG or JPG both work. Keep each under ~500 KB for fast loading.
